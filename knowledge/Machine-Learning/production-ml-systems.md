# Production ML systems | Machine Learning | Google for Developers

This module outlines the broader ecosystem needed to operate machine learning in production, emphasizing that the model constitutes only a small fraction of the overall system. It guides practitioners in selecting between static and dynamic paradigms for both training and inference based on product needs and data behavior. The content focuses on robust data handling, deployment testing, monitoring, and asking diagnostic questions to uncover and mitigate flaws. Building on foundational ML concepts, it centers on practical deployment and maintenance in real-world scenarios.
- Recognize production ML components beyond the model, including data collection, validation, and feature engineering.
- Choose between static vs. dynamic training and decide retraining cadence to match data drift and product risks.
- Select static vs. dynamic inference with awareness of latency, freshness, and consistency trade-offs.
- Decide where to perform data transformations (training vs. serving) to prevent train–serve skew.
- Implement deployment testing (unit, integration, canary) to validate end-to-end behavior.
- Monitor data, models, and pipelines for drift, outages, and performance regressions.
- Ask targeted operational questions to identify failure modes and reliability risks.
