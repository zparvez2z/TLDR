# Embeddings | Machine Learning | Google for Developers

This module introduces embeddings as dense, lower-dimensional representations that address the limitations of sparse encodings like one-hot vectors. It explains how embeddings reduce parameter counts, data requirements, computation, and memory, while capturing semantic relationships between items (for example, hot dogs and shawarmas being more similar than hot dogs and salads). The content covers the concept of embedding spaces, static versus contextual embeddings, and practical ways to obtain and visualize embeddings (such as word2vec). Prerequisites include familiarity with linear regression, categorical data handling, and neural networks.
- One-hot encoding leads to very large input vectors and M×N weight counts in the first layer, inflating data, compute, and memory needs.
- Dense embeddings capture semantic similarity, overcoming the lack of meaningful relations in one-hot encodings.
- Clear distinction between encoding (initial numeric representation) and embedding (learned dense representation).
- Coverage of embedding space intuition, static embeddings, and contextual embeddings.
- Practical approaches to obtaining embeddings and visualizing them (e.g., word2vec).
- Benefits for efficiency, model size, and on-device ML feasibility.
- Includes interactive exercises and knowledge checks; assumes foundational ML knowledge.
