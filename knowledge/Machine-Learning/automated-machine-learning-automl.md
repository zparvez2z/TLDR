# Automated Machine Learning (AutoML) | Google for Developers

AutoML automates repetitive tasks in the machine learning development cycle, including feature engineering, algorithm selection, and hyperparameter tuning. It reduces the need for specialized coding and experimentation, making model building faster and more accessible while allowing teams to focus on data and problem formulation. This module explains AutoML’s benefits and limitations, common patterns, and practical guidance for getting started, assuming familiarity with foundational ML concepts. It highlights automation opportunities across data engineering, training, and analysis/evaluation with validation and test sets.
- Automates feature engineering, feature selection, algorithm choice, and hyperparameter tuning.
- Benefits: faster iteration, reduced repetitive work, lower need for specialized skills; Limitations: requires understanding of data/metrics and oversight.
- Common AutoML patterns cover data engineering, training, and analysis stages.
- Enables teams without dedicated data scientists to prototype and build models more feasibly.
- Prerequisites include intro to ML, linear regression, handling numerical and categorical data, and datasets/generalization/overfitting.
- Module duration ~30 minutes with sections: Introduction, Benefits and limitations, Getting started.
