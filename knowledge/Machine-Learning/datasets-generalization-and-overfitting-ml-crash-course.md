# Datasets, generalization, and overfitting | Machine Learning | Google for Developers

This module explains how to prepare and assess datasets so machine learning models generalize well and avoid overfitting. It covers identifying dataset characteristics, detecting and handling unreliable or missing data, and improving labels (including human-rated labels). Learners are guided through proper dataset splitting (training, validation, test), consistent data transformations, and techniques to prevent overfitting such as L2 regularization and early stopping. The module also teaches how to interpret loss curves to diagnose convergence and bias–variance issues.
- Data quality typically impacts model performance more than algorithm choice; expect significant effort in data preparation and feature engineering.
- Identify data and dataset characteristics and common sources of unreliability; decide when to discard vs. impute missing data.
- Differentiate direct vs. derived labels and improve label quality, especially for human-rated labels.
- Address imbalanced datasets and apply transformations consistently across splits.
- Split data into training, validation, and test sets properly to avoid leakage and biased evaluation.
- Understand generalization and overfitting; mitigate with model complexity control, L2 regularization (lambda), and early stopping.
- Interpret loss curves to spot convergence, high bias vs. high variance, and signs of overfitting.
