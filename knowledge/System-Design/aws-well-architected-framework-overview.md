The AWS Well-Architected Framework provides best-practice guidance to design, evaluate, and improve cloud workloads on AWS. It is organized around six pillars—operational excellence, security, reliability, performance efficiency, cost optimization, and sustainability—to help teams build secure, high-performing, resilient, and efficient systems. Customers can extend evaluations with domain-specific Lenses, apply targeted Guidance for specific use cases, and use the free AWS Well-Architected Tool to identify high-risk issues and track remediation. A global network of AWS Well-Architected Partners is available to support reviews and improvements.
- Six pillars: operational excellence, security, reliability, performance efficiency, cost optimization, and sustainability
- AWS Well-Architected Tool (free in the AWS Management Console) for regular workload assessments and improvement tracking
- Domain-specific Lenses for areas like machine learning, analytics, serverless, HPC, IoT, SAP, media streaming, gaming, hybrid networking, and financial services
- Guidance materials focus on specific use cases and scenarios, complementing the broader Framework and Lenses
- Evaluation driven by foundational questions to align architectures with AWS best practices and scalable design patterns
- Partner Program offers expert assistance to analyze and review applications
- Emphasis on faster delivery, reduced risk, and informed decision-making using a consistent approach
