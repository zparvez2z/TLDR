# What is a REST API? Red Hat Red Hat AI icon Red Hat Enterprise Linux icon Red Hat OpenShift icon Red Hat Ansible Automation Platform icon

This article explains what a REST API is and how it differs from general APIs by adhering to the REST architectural style. It clarifies that an API serves as a contract and mediator between consumers and providers, enabling secure and controlled access to resources. The piece highlights common data representation formats—especially JSON—and emphasizes the importance of HTTP headers and parameters in conveying request and response metadata. It also outlines the key REST constraints that make an API RESTful, including client-server separation, statelessness, cacheability, and a uniform interface.
- REST is a set of architectural constraints for web APIs, not a protocol or standard.
- APIs act as contracts/mediators, enabling integration while enforcing security, control, and authentication.
- RESTful responses transfer a representation of resource state via HTTP, commonly using JSON (language-agnostic and human/machine readable) among other formats.
- HTTP headers and parameters carry crucial metadata, authorization, URIs, caching directives, cookies, and status information.
- To be RESTful, APIs must be client-server, stateless, cacheable, and provide a uniform interface.
