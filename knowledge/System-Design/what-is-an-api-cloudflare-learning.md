# What is an API?

This article explains what an application programming interface (API) is and how it enables software programs to communicate and reuse functionality. It defines core concepts like API calls (requests), endpoints, schemas, and integration, using a simple traffic-checking example. It also distinguishes web APIs from other types and compares SOAP and REST approaches. Finally, it outlines common API security risks and essential protections such as rate limiting, DDoS mitigation, authentication (including mTLS), and schema validation.
- An API is a set of rules enabling one software program to transmit data to and interact with another, reducing redundant development work.
- API calls must follow the API’s schema and be sent to a specific endpoint (URL) to receive valid responses.
- API integration connects multiple applications to share functionality and synchronize data.
- Web APIs power most modern Internet applications and architectures, including JAMstack and microservices.
- SOAP APIs use the SOAP protocol; REST APIs follow a flexible architectural style and can use any protocol, with REST being most common today.
- API security measures include rate limiting, DDoS protection, strong authentication (e.g., mutual TLS), and schema validation to prevent misuse and unexpected behavior.
